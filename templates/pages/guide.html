User Guide

Input Formatting

Minor Allele Frequencies (MAFs)

From Data

This option is only recommended for datasets with more than 250 samples. We also recommend that you enable MCMC when using this option, to better approximate the true MAFs present in your sampled population.

From File

This option is recommended if validated MAFs are available for your dataset. The MAFs file should be formatted like this sample. In short, either a .csv or tab-delimited file, with 4 columns corresponding to an index (or barcode position id), the major allele, the minor allele and the minor allele frequency, respectively. All lines that start with # will be ignored.

From pf3k

This option is recommended for smaller datasets, where validated MAFs are not available. For this option, you must choose the geography relevant to your samples (listed in the dropdown), as well as submit a file specifying the genomic positions of your SNP barcodes. The SNP loci file should be formatted like this sample.

SNP Barcodes

The SNP barcodes file should be formatted like this sample. In short, a tab-delimited file with 2 columns, corresponding to the sample id and barcode, respectively. Like the SNP loci file, all lines starting with # will be ignored.


Output Formatting

The output is a .csv file with each row corresponding to either a sample (in the same order as the input) or a SNP loci position, with the sample rows listed first and the SNP loci listed second; and with columns corresponding to ids and predictions. In the case of the sample rows, the predictions are COI. In the case of the SNP loci rows, the predictions are MAFs (which may or may not be different than input MAFs depending on whether MCMC approximation was enabled).
